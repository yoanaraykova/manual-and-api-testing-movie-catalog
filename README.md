# Manual QA Portfolio Project

## Description

This project demonstrates my skills as a **Manual QA Specialist**, focusing on both **web application testing** and **API testing with Postman**.

The project is based on a **Movie Catalog application**, with the main goal to ensure the app functions correctly according to defined use cases. The work includes **manual test cases**, **bug reports**, and **API test requests**.

## Features / Use Cases

The Movie Catalog Web App allows users to:

* Register and log in.
* Edit their profile information.
* Add, edit, delete, and view movies.
* Mark movies as watched or unwatched.
* Navigate easily between Landing, Home, and Movie pages.

Test cases cover the following use cases:

1. Landing/Home Page functionality for registered and unregistered users.
2. User registration and login.
3. Profile management.
4. Adding movies.
5. Movie management (view, edit, delete, mark watched/unwatched).

API endpoints tested include:

* **User Endpoints:** Register and Authenticate users.
* **Movie Endpoints:** Create, Edit, Delete movies.
* **Catalog Endpoint:** List all movies.

## How to Use

1. Clone the repository:

```bash
git clone https://github.com/your-username/ManualQA-Portfolio.git
```

2. Import **PostmanCollection.json** into Postman:

   * Open Postman → Import → File → select `PostmanCollection.json`.
   * Run the requests to test API endpoints.
3. Open **Test-Management-and-Bug-Tracker.xlsx** to review documented test cases, bug reports, and screenshot of expected behavior.
